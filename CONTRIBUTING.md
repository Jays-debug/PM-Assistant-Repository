# Contributing

Thank you for contributing to Fleet Management Platform.

## Development Workflow

1. Pull latest changes.
2. Create a feature branch.
3. Develop your feature.
4. Test locally.
5. Commit with clear messages.
6. Push to GitHub.
7. Create Pull Request.

---

## Commit Style

Examples

```
Add PM Calendar

Fix Vehicle Filter

Improve Dashboard

Refactor Notification Service
```

---

## Coding Standards

- Keep code readable
- Comment important logic
- Follow existing project structure
- Test before commit

---

## Branch Naming

```
feature/dashboard

feature/calendar

bugfix/login

hotfix/api
```
